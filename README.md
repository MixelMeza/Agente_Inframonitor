# InfraMonitor Remote Agent

Lightweight Java agent (zero dependencies) that connects to the InfraMonitor backend and
executes network checks **from inside a remote network** — no inbound ports required on the
agent host.

## How it works

```
[ Remote Network ]          [ Internet ]         [ InfraMonitor Backend ]
  Agent (JAR)  ──────────── WebSocket ──────────▶  ws://your-backend/ws/agents
  ↓ executes checks locally                         ↑ sends check commands
  ↑ returns results                                 ↑ receives results
```

## Build

Requires JDK 11+ and Maven:

```bash
cd inframonitor-agent
mvn package -q
# Produces: target/inframonitor-agent.jar
```

## Deploy

1. **Create an agent in InfraMonitor** (as admin):

   ```
   POST /api/agents
   { "name": "office-network", "description": "Main office LAN" }
   ```

   Save the returned `apiKey` — it is shown only once.

2. **Copy the JAR** to the remote machine (any machine with JRE 11+):

   ```bash
   scp target/inframonitor-agent.jar user@remote-server:/opt/inframonitor/
   ```

3. **Run the agent**:

   ```bash
   java -jar inframonitor-agent.jar \
     --server=ws://your-backend-host:9875/ws/agents \
     --name=office-network \
     --key=ima_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```

4. **Add a service** in InfraMonitor using type `REMOTE_AGENT`, select the agent name, and
   configure the target host/port to check.

## Options

| Option | Default | Description |
|--------|---------|-------------|
| `--server` | required | Backend WebSocket URL |
| `--name` | required | Agent name (must match what was created in backend) |
| `--key` | required | API key generated by backend |
| `--heartbeat` | 30 | Seconds between heartbeat messages |
| `--reconnect-delay` | 5 | Seconds to wait before reconnecting after disconnect |

## Run as a systemd service (Linux)

Create `/etc/systemd/system/inframonitor-agent.service`:

```ini
[Unit]
Description=InfraMonitor Remote Agent
After=network.target

[Service]
Type=simple
User=monitor
ExecStart=/usr/bin/java -jar /opt/inframonitor/inframonitor-agent.jar \
  --server=ws://your-backend:9875/ws/agents \
  --name=office-network \
  --key=ima_xxxx
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
systemctl enable inframonitor-agent
systemctl start inframonitor-agent
```

## Supported check types (executed locally on the agent host)

| Type | What it does |
|------|-------------|
| PING | `InetAddress.isReachable()` |
| TCP | Socket connect to host:port |
| HTTP | HEAD request, checks HTTP 2xx/3xx |
| DNS | DNS resolution of hostname |
